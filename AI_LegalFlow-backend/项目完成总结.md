## 🚀 快速开始

### 1. 环境要求
- JDK 17+
- Maven 3.6+
- 8GB+ 内存

### 2. 启动应用
```bash
cd AI_LegalFlow-backend
mvn spring-boot:run
```

### 3. 访问应用
- 应用地址：http://localhost:8080
- H2控制台：http://localhost:8080/h2-console
- API测试：参考 POSTMAN测试文档.md

### 4. 测试流程
```bash
# 1. 测试知识库检索
curl "http://localhost:8080/api/knowledge/search?query=竞业限制&topK=5"

# 2. 创建任务
curl -X POST http://localhost:8080/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"message":"劳动合同中的竞业限制条款有哪些风险？","taskType":"LEGAL_QA"}'

# 3. 查询任务
curl http://localhost:8080/api/tasks/{taskId}
```

## 🎯 核心功能演示

### 流程1：法律问答
```
用户提问 → 协调者Agent → 法规检索Agent → 报告生成 → 返回结果
```

**测试问题：**
- "竞业限制条款有哪些风险？"
- "违约金的法律规定是什么？"
- "试用期最长期限是多久？"

### 流程2：合同审查
```
上传合同 → 条款抽取 → 法规检索 → 风险识别 → 建议生成 → 报告生成
```

**功能：**
- 自动识别合同条款
- 匹配相关法律法规
- 识别法律风险点
- 生成修改建议
- 输出完整报告

## 🔧 技术栈

### 后端
- **框架**: Spring Boot 3.2.0
- **AI**: Spring AI 1.1.6
- **数据库**: H2 Database (开发), PostgreSQL (生产)
- **文档解析**: Apache Tika
- **PDF生成**: iText 7
- **构建工具**: Maven

### AI模型
- **主模型**: 阿里云百炼 - 通义千问 (qwen-plus)
- **接口**: OpenAI兼容API

## 🔐 安全配置

### API Key管理
- 开发环境: 直接配置在 application.yml
- 生产环境: 使用环境变量或密钥管理服务

```yaml
spring:
  ai:
    openai:
      api-key: ${ALIYUN_API_KEY:your-api-key}
```

### 数据安全
- H2数据库: 仅用于开发
- 生产环境: 使用PostgreSQL + SSL
- 敏感数据: 加密存储
